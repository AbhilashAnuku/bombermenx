What is BomberMen-X?

BomberMen-X is a four-player real-time arena game with a server-authoritative simulation. Players place bombs on a tile grid, walls break, opponents are eliminated, power-ups are collected, and the last bomber standing wins. The visual identity follows an Indian-festival mandala palette: peacock teal, turmeric yellow, henna pink and gold on an aubergine background.

The project was chosen because a multiplayer arena game forces almost every architectural quality attribute into the open — latency, fairness, determinism, concurrency, modifiability, testability. It gives the team a realistic surface on which to demonstrate the techniques covered in the Software Architecture & Development module without inventing a synthetic problem.

The system is implemented as a three-module Maven reactor in pure Java 17. The shared bomberman-core library defines the domain entities and the wire DTOs; bomberman-server is a single-process Netty WebSocket server that holds the only mutable game world and advances it at sixty ticks per second; bomberman-client is a JavaFX desktop application that renders snapshots and ships input upstream. No browser, no Spring, no exotic frameworks — a deliberately small stack so the architecture decisions are visible.

What is being submitted

The submission consists of four artefacts. Each is reproducible from this portal in under five minutes.

  1. Architecture Report (canonical document). A single-page HTML report that walks an examiner from goals and constraints through building blocks, runtime view, deployment, quality requirements, decisions and risks. This is the document that defines what was built and why. → architecture-report-en.html
  2. arc42 Specification (long form). The 22-section arc42 specification covering every level of the architecture with diagrams and tables. The Architecture Report above is the executive summary; this is the underlying detail. → code-explanation/architecture-spec-arc42 · DOCX in exports/
  3. Working Prototype (the running system). A Maven reactor under ../src/ that builds clean (mvn clean test → 11 green tests across three modules), a server jar that listens on port 8080, and a JavaFX client launched by infra/scripts/run-client.cmd. → RUN_GUIDE covers prerequisites, build, and run, end-to-end
  4. Defence Pack (DOCX exports). Six markdown documents exported as Word files so the team can edit final wording in Google Docs or Word before the viva: arc42 spec, requirements traceability, course context, systems architecture, run guide, learning guide. No PDF is generated — DOCX is the only output, intentionally. → exports/ folder (six .docx files plus an index)

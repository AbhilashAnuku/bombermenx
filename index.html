Overview

Überblick

BomberMen-X is implemented as a Maven three-module Java 17 reactor. The shared bomberman-core library defines the domain entities and the wire DTOs. bomberman-server is a single-process Netty WebSocket server that owns the only mutable game world and advances it at sixty ticks per second. bomberman-client is a JavaFX desktop application that renders snapshots and ships input upstream. The strict dependency direction — both endpoints import the shared core — makes data-transfer-object drift structurally impossible.

The architecture is the smallest configuration that satisfies five explicit quality goals: determinism (the simulation is single-threaded and replayable), fairness (the server is the sole authority over the game state), clarity (every wire message is a typed JSON envelope), portability (a portable JDK 17 plus Maven 3.9 is the entire toolchain), and demonstrability (the prototype starts with a single command).

BomberMen-X ist als Maven-Mehrmodulreaktor in reinem Java 17 umgesetzt. Die gemeinsame Bibliothek bomberman-core definiert die Domänen-Entitäten und die Wire-DTOs. bomberman-server ist ein Netty-WebSocket-Server in einem einzigen Prozess, der den einzigen veränderbaren Spielzustand hält und mit sechzig Ticks pro Sekunde fortschreibt. bomberman-client ist eine JavaFX-Desktop-Anwendung, die Snapshots rendert und Eingaben stromaufwärts überträgt. Die strikte Abhängigkeitsrichtung — beide Endpunkte importieren den gemeinsamen Kern — schließt eine Drift der Datenübertragungsobjekte strukturell aus.

Die Architektur ist die kleinste Konfiguration, die fünf explizite Qualitätsziele erfüllt: Determinismus, Fairness, Klarheit, Portabilität und Demonstrierbarkeit.

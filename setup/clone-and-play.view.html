Clone and play in 5 minutes

Two paths to a playable arena: a full Dockerised stack with the desktop client, or a zero-install browser client. Either one fits inside a coffee break.

1. Prerequisites

Install once. Skip anything you already have.

  • JDK 21 (Eclipse Temurin) — required for the desktop client and Maven builds.
    Windows: scoop install temurin21-jdk · macOS: brew install --cask temurin · Ubuntu/Debian: sudo apt install temurin-21-jdk (Adoptium APT repo)
  • Maven 3.9 — multi-module build orchestrator.
    Windows: scoop install maven · macOS: brew install maven · Ubuntu/Debian: sudo apt install maven
  • Docker — runs the server, Postgres 16, Redis 7.
    Windows/macOS: install Docker Desktop · Ubuntu/Debian: sudo apt install docker.io docker-compose-plugin
  • Git — clone the repository.
    Windows: scoop install git · macOS: brew install git (or Xcode CLT) · Ubuntu/Debian: sudo apt install git

2. From clone to playable in three commands

1Clone the repository

git clone <repo URL> BomberMan-X

Replace <repo URL> with the repository URL handed out by the team. The clone takes a few seconds.

2Bring up the server stack

cd BomberMan-X && docker compose up -d --build

Builds the server image and starts bomb-server, Postgres 16, and Redis 7 on the internal bxnet bridge. First build pulls base images and runs mvn -B package inside the builder stage; expect 2–4 minutes on a warm Maven cache. Tail the logs until you see WebSocket server listening on ws://0.0.0.0:8080/ws.

3Launch the desktop client (second terminal)

mvn -B -pl bomb-core,bomb-client -am package && mvn -B -pl bomb-client javafx:run

The first Maven invocation builds bomb-core and bomb-client together (the -am flag pulls in dependent modules). The second launches the JavaFX 21 client, which goes straight to the age-gate dialog and then the main menu. Hit JOIN LOBBY and you are in a match against three bots within eight seconds of warmup.

3. Or: zero-install web client

If you only want to play (no JavaFX, no Maven), open the bundled browser client directly from the repository checkout. It speaks the same wire protocol as the desktop client.

OSCommand
Windowsstart presentation\play.html
macOSopen presentation/play.html
Linuxxdg-open presentation/play.html

Click CONNECT, then JOIN QUEUE. The server URL defaults to ws://localhost:8080/ws; change it from the field if you point at a remote server.

5. Troubleshooting

SymptomCauseFix
Port 8080 already in use Another process owns 8080 on the host Stop the other process, or remap the host port in docker-compose.yml (for example "18080:8080") and connect to ws://localhost:18080/ws.
JavaFX client fails with “module javafx.controls not found” (Linux) OpenJFX runtime modules not on the JDK 21 in use, or Maven did not resolve them on cold cache Install openjfx for the system JDK (sudo apt install openjfx) or use an Eclipse Temurin 21 build with bundled FX; then mvn -U dependency:resolve and retry mvn javafx:run.
Gamepad not detected Pad plugged in after the menu scene was shown; JInput only rescans on scene change Press Esc to bounce back to the menu and re-enter, or reconnect the pad before launching.
First boot of bomb-server hangs on Postgres Postgres 16 first-run initialisation is slow on a cold disk Wait 30–60 seconds. The server’s health check polls every 5 s and starts as soon as pg_isready succeeds. Run docker compose logs -f bomb-server to watch progress.

If anything else goes wrong, the most useful single command is docker compose logs -f bomb-server. The slf4j output names the failing collaborator and the line is usually enough to localise the fix.

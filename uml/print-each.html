Class Diagram — bomb-server

BombServerApplication is the composition root. It builds the WebSocket server, the session registry, the match manager, the auth providers, the lobby service, and the moderation stack. Implementations (e.g. DevAuthProvider, GoogleAuthProvider) sit behind the AuthProvider interface — Liskov-clean.

UML / CLASS · bomb-client src: uml/class-diagram-client.svg

Class Diagram — bomb-client

The JavaFX side. ClientLauncher bootstraps the age gate and the SceneRouter, which swaps between menu, lobby, rankings, and the in-game arena. ArenaView drives the render pipeline (renderer + particles + camera shake + post-FX) and pulls input through the gamepad poller; GameClient is the single WebSocket adapter every scene shares.

Package Diagram

Three columns — one per Maven module — with dependency edges between them. Both runtime modules depend on bomb-core; the dashed line between client and server is the explicit non-dependency that the layering rule enforces. Inside core, sub-packages stack from net / net.dto at the top down through sim, entity, world, and geom, with GameConfig at the root.

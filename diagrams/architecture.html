System Architecture

BomberMen-X is a three-tier real-time multiplayer system. The client tier (JavaFX desktop, web HTML, and a deferred 3D spectator) renders the game and forwards player intent. The shared bomberman-core module provides the domain entities, the deterministic simulation, and the wire protocol — re-used unchanged on both sides. The server tier is built on Netty and owns the authoritative simulation, lobby, matchmaker and match sessions, with Postgres and Redis as durable and ephemeral sidecars.

Key decisions

  • Authoritative server. Clients send intents only; the server validates, simulates and broadcasts snapshots.
  • One shared module. Domain entities and wire DTOs live in bomberman-core so client and server stay byte-compatible.
  • 60 Hz tick, 20 Hz broadcast. The simulation runs fixed-step; snapshots are delta-encoded against the last ACKed tick.
  • Stateless tier. Server instances hold only in-flight match state; durable state lives in Postgres, ephemeral state in Redis.
